# 拾光集时间轴相册设计

## 目标

将“拾光集”改为照片优先的时间轴相册。进入该空间后，用户无需打开详情即可查看图片缩略图，并可直接在图片卡片上修改记录日期。

## 已确认界面

- 主笔记区在进入 `memory` 空间后显示左侧日期时间轴。
- 同一日期的图片以统一近方形比例的三列照片墙排列，第二张仅轻微下移以保留自然感。
- 图片使用 `object-fit: cover`，每张卡片保留标题小标签；点击卡片仍打开现有沉浸阅读器。
- 管理员悬浮图片时显示右上角编辑日期按钮；点击后显示原生日期输入框，改变日期立即保存并按新日期重排。
- 访客只能看照片与日期，不显示日期编辑按钮。
- 手机上的相册为两列、同尺寸布局；时间轴缩到左侧细线且不产生横向溢出。

## 数据与安全

- 继续使用已有 `recordDate` / `record_date` 字段，不变更数据库 schema。
- 仅 `space === "memory"` 且 `type === "图片"` 的内容加入照片时间轴；其他拾光集内容保留普通笔记卡片，避免视频和文本丢失。
- 日期修改使用现有 `persist()`；云端仍受管理员写入的 RLS 规则限制。
